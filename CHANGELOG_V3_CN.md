# v3 更新说明：严格时效、准确来源、5 类分类

本次更新已经把日报面板改成你要求的逻辑。

## 1. 严格时效

默认只展示最近 **36 小时** 内、且能确认发布时间的信息。

- 超过 36 小时的信息不会进入“当前有效信息要点”。
- 没有发布时间、无法解析发布时间、网页只显示静态旧内容的信息不会进入重点卡。
- 被过滤信息只会以汇总形式出现在“时效过滤说明”和“受限/失败来源”中。

如需放宽或收紧，在 GitHub 仓库的 `.github/workflows/daily-update.yml` 里改：

```yaml
FRESHNESS_WINDOW_HOURS: "36"
```

比如改成 `24`，就是只看最近 24 小时。

## 2. 每条信息都有准确来源

新版不再只显示“来源名称”，而是在每个统一要点后附上所有原始链接：

- 原始标题
- 来源名称
- 发布时间
- 距当前多久
- 原帖/原文 URL

如果多个来源反映同一件事，系统会合并成同一个“统一要点”，然后在该要点下列出全部原帖链接。

## 3. 面板只保留 5 类

1. 竞品动态：竞品官方网站、官方博客、新闻室、更新日志、官方论坛。
2. 社交媒体：竞品官方社交账号内容，例如官方 X/Twitter、YouTube、TikTok、LinkedIn。
3. reddit讨论：Reddit 用户发帖和评论讨论。
4. 政策风险：政府、监管机构、官方政策公告和合规页面。
5. 第三方网站：媒体、评测站、商店评论、论坛、搜索页和其他来源。

## 4. 已移除增长运营行动建议

新版只做信息观察、分类归档、时效判断、来源追踪和受限来源说明。

## 5. Reddit 访问兜底仍保留

Reddit 抓取顺序：

```text
Reddit OAuth API → public JSON → RSS → old.reddit HTML → manual_inputs.csv 手工补充
```

如果 Reddit 仍然失败，建议在 GitHub Secrets 里配置：

```text
REDDIT_CLIENT_ID
REDDIT_CLIENT_SECRET
REDDIT_USER_AGENT
```
